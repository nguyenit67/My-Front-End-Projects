(RE)STYLING

This tab component is created with just HTML and CSS - no Javascript. The tabs shown here are left mostly as-is from a client website. They do not have to be left square as shown - they can be styled to take on any appearance (see comments in the CSS). Also, have a look at this site for inspiration (note that they use ul/li in their HTML structure as well as a little Javascript to achieve the final result).

The client for which this component was developed needed a border around eveything as shown here. The border is also left as-is because styling with the border was initially more difficult than styling with solid colors (see the related blog post about the problem encountered when using a border).

HANDLES DYNAMIC CONTENT AND KEEPS IT WITHIN THE DOCUMENT FLOW

Each tab's section grows vertically with the content. Go ahead - click on each of the other tabs to see. Along with the fact that each tab section grows as needed, a key feature of this tab component is that it DOES NOT USE ABSOLUTE POSITIONING. Other pure HTML/CSS Tab components that were tried/researched used absolute positioning and so when the length of the tab content area grew as variable length dynamic content was filled in, the tab component extended below the bottom of the page because the absolute positioning messed up the flow of elements on the page such that the page length would not grow to enclose the tab.

BACKGROUND OF HOW THE COMPONENT WAS DEVELOPED

This component actually started when another component on CodePen was tried and proved not to work as needed. See the blog post describing the development of this component and the nuances that were encountered along the way including the absolute positioning problem, the 1px border problem, a discussion on the overuse of ul/li, and an explanation of how Flexbox provided the functionality needed to ultimately make this component work without needing any assistance from Javascript. The background on each mediaitem is colored just to show it's extent since no thumbnails are included here.

IMPLEMENTATION

While quick implementation cannot be guaranteed, because of the styling chosen and the name-spacing employed, this component should be able to be dropped into existing HTML and used with very minimal changes other than styling the appearance and changing the max-width on the #tabscontainer.

Discussion - Flexbox

This media tab was also used in the client's website and is left here as an example. The multiple divs in the HTML that are of CLASS=mediaitem could, of course, be replaced with a single mediaitem div within a query call to the server that would then automatically create multiple mediaitem divs for all items selected by the query. Note that Flexbox is also used to lay out the mediaitem divs.
